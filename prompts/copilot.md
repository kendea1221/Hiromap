GitHub Copilot用プロンプト
このプロンプトは、App.jsx や主要なコンポーネントを作成する際に使用してください。

プロンプトの内容:

React (Vite) を使用して、観光レコメンドサービス「HiroMap」のプロトタイプを作成してください。以下のUI構成と機能を含めてください。

1. UI構成:

ベースレイヤー: 全面に Google Maps 風の地図を表示（背景画像やダミーの地図ライブラリでOK）。

チャットUI: 画面下部またはサイドに配置される、シンプルで機能的なチャットインターフェース。

地図上のピン: 観光スポットを示すピン。最新の投稿ほど「光る（明るい色）」エフェクトを持つようにスタイルを適用。

2. チャットの対話フロー（デモ用）:

AIは「シンプルで機能的なツール」としてのトーン。

最初の質問: 「広島へようこそ。今の滞在時間はどれくらいですか？」

ユーザーが時間を入力（または選択）すると、現在の環境データ（例: 28°C / 湿度 65% ※ダミーデータでOK）を表示し、それに適したスポットを2〜3個提案する。

提案文の例: 「現在は少し蒸し暑いため、涼しい室内で歴史を感じられる〇〇ギャラリーはいかがですか？」

3. 主要機能のモック:

スポット提案をクリックすると、地図上のその地点へフォーカスする。

「今のログを残す」ボタンを押し、一言と写真を投稿するシミュレーション（投稿後、地図上に新しい明るいピンが追加される）。

4. 技術スタック・デザイン:

Tailwind CSS を使用してモダンで清潔感のあるデザインに。

アイコンは lucide-react を使用。

状態管理は useState で完結させ、複雑なバックエンド接続は今のところ不要（フロントエンドのみで完結）。

5. 独自ポイントの強調:
